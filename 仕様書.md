# 棚卸し在庫管理アプリ 仕様書

## 1. 概要 / 目的

物販・在庫管理における「棚卸し（実地棚卸）」を楽にすることを主目的とした Web アプリ。
商品ごとに **在庫数・販売価格・仕入価格・個数** を管理し、定期的な棚卸しで
「帳簿上の在庫」と「実際に数えた在庫」の差異を素早く把握できるようにする。

- 想定利用者: 個人の物販事業者 / 小規模店舗（まずは自分自身の運用が対象）
- 利用シーン: 普段の在庫管理 ＋ スマホを手に棚の前で行う棚卸し作業
- ゴール: 在庫評価額・想定粗利が自動で見え、棚卸しの差異チェックが数分で終わる状態

## 2. 技術構成（フレームワーク）

| レイヤー | 技術 | 役割・備考 |
|---|---|---|
| フロントエンド / API | Next.js（App Router）+ TypeScript | 画面と API ルート |
| デザイン | Tailwind CSS | スマホ最適化しやすい |
| データベース | Supabase（PostgreSQL） | 商品・棚卸しデータの保存 |
| 認証 | Supabase Auth | メール＋パスワードログイン |
| ストレージ | Supabase Storage | 商品画像の保存 |
| アクセス制御 | Supabase RLS（Row Level Security） | ログインユーザー自身のデータのみ参照・編集可 |
| バーコード読取 | html5-qrcode（またはブラウザの BarcodeDetector API） | スマホカメラで JAN コード読取 |
| CSV 入出力 | papaparse | 一括登録・書き出し |
| グラフ | recharts | 集計・分析の可視化 |
| ホスティング / 公開 | Vercel | 無料枠でデプロイ |

> いずれも無料枠内で開発・公開が可能。決済（Stripe）は当面不要。

## 3. 機能要件

優先度: **P0 = MVP（最初に必須）** / **P1 = 棚卸しの核心** / **P2 = 拡張（後回し可）**

### 3.1 認証（P0）
- 新規登録 / ログイン / ログアウト
- 未ログイン時は商品・棚卸し画面へアクセスできずログイン画面へリダイレクト

### 3.2 商品マスタ管理（P0）
- 商品の登録・一覧・編集・削除（CRUD）
- 1 商品あたり以下を保持:
  - 商品名 / 管理番号（SKU・JAN コード） / カテゴリ
  - 仕入価格（円） / 販売価格（円） / 在庫数（個数）
  - 保管場所（棚番） / 仕入先 / メモ / 商品画像
- 一覧の検索（商品名・管理番号）
- 一覧の並び替え・フィルタ（カテゴリ別 / 在庫が少ない順 など）

### 3.3 集計ダッシュボード（P0）
- 総在庫評価額 ＝ Σ（仕入価格 × 在庫数）
- 総想定粗利 ＝ Σ（（販売価格 − 仕入価格）× 在庫数）
- 総商品点数 / 総在庫数
- カテゴリ別の在庫評価額（グラフ）

### 3.4 棚卸しモード（P1・核心機能）
- 棚卸しセッションの作成（例:「2026年8月 棚卸し」）
- セッション開始時、その時点の在庫数を「帳簿在庫」としてスナップショット保存
- 各商品について「実地在庫（実際に数えた数）」を入力
- **差異（実地 − 帳簿）を自動計算し、差異のある商品をハイライト**
- クイック入力: ＋ / − ボタン・テンキーで素早くカウント
- 数え済み / 未カウントの区別
- セッション完了時、差異レポート（差異のある商品一覧）を表示
- 過去の棚卸しセッションを履歴として閲覧できる

### 3.5 バーコード / QR スキャン（P1・核心機能）
- スマホカメラで JAN コードを読み取り、該当商品を即座に呼び出して数量入力
- 棚卸しモード内から起動して「歩きながら数える」を実現
- 該当商品が未登録なら、その場で新規登録に進める

### 3.6 入出庫・在庫変動履歴（P2）
- 入庫（仕入れ）/ 出庫（販売）を記録すると在庫数を自動増減
- 変動ログ（いつ・何を・何個・種別）を保持

### 3.7 CSV インポート / エクスポート（P2）
- 既存在庫を CSV で一括登録
- 商品一覧を CSV で書き出し（他サービスとのデータ連携用）

### 3.8 その他拡張（P2）
- 在庫アラート（設定数を下回ったら一覧で警告表示）
- 在庫金額の推移グラフ
- 複数ユーザーでの共有（将来）

## 4. データモデル（テーブル設計）

すべてのテーブルに `user_id`（auth.users への外部キー）を持たせ、RLS で本人のデータのみアクセス可とする。

### products（商品）
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | 商品 ID |
| user_id | uuid (FK) | 所有ユーザー |
| name | text | 商品名 |
| management_code | text | 管理番号・JAN コード |
| category | text | カテゴリ |
| purchase_price | integer | 仕入価格（円） |
| selling_price | integer | 販売価格（円） |
| stock_quantity | integer | 在庫数（帳簿在庫） |
| location | text | 保管場所・棚番 |
| supplier | text | 仕入先 |
| image_url | text | 商品画像 URL（Supabase Storage） |
| memo | text | メモ |
| created_at | timestamptz | 作成日時 |
| updated_at | timestamptz | 更新日時 |

### stocktakes（棚卸しセッション）
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | セッション ID |
| user_id | uuid (FK) | 所有ユーザー |
| name | text | 棚卸し名 |
| status | text | `in_progress` / `completed` |
| started_at | timestamptz | 開始日時 |
| completed_at | timestamptz | 完了日時 |

### stocktake_items（棚卸し明細）
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | 明細 ID |
| stocktake_id | uuid (FK) | 対象セッション |
| product_id | uuid (FK) | 対象商品 |
| book_quantity | integer | 帳簿在庫（開始時スナップショット） |
| actual_quantity | integer | 実地在庫（数えた数） |
| counted | boolean | 数え済みフラグ |
| difference | integer | 差異（actual − book、計算 or 保存） |

### stock_movements（在庫変動履歴・P2）
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | ログ ID |
| user_id | uuid (FK) | 所有ユーザー |
| product_id | uuid (FK) | 対象商品 |
| type | text | `purchase`（仕入）/ `sale`（販売）/ `adjustment`（棚卸調整） |
| quantity_change | integer | 増減数（＋／−） |
| note | text | 備考 |
| created_at | timestamptz | 記録日時 |

### 計算で求める値（テーブルには持たない）
- 在庫評価額 ＝ purchase_price × stock_quantity
- 想定粗利 ＝ (selling_price − purchase_price) × stock_quantity

## 6. 非機能要件

- **レスポンシブ / PWA 対応**: スマホを手に棚の前で操作できること（棚卸しの核心）
- **セキュリティ**: Supabase RLS により、ユーザーは自分のデータのみアクセス可能
- **認証**: 未ログイン時は保護ページへアクセス不可
- **費用**: Supabase・Vercel の無料枠内で運用（初期費用ゼロ）
- **金額表記**: 円・整数で扱う（小数は扱わない）

## 7. 開発の進め方（ステップ）

Claude Code には、以下の順で「1 ステップずつ」進めるよう依頼する。

1. **ステップ1**: 認証（Supabase Auth）＋ 商品 CRUD ＋ 商品一覧
2. **ステップ2**: 集計ダッシュボード ＋ 検索・並び替え・フィルタ
3. **ステップ3**: 棚卸しモード ＋ バーコードスキャン（核心）
4. **ステップ4**: CSV 入出力 ＋ PWA 対応 ＋ 仕上げ・Vercel デプロイ

各ステップごとに動作確認してから次へ進む。
